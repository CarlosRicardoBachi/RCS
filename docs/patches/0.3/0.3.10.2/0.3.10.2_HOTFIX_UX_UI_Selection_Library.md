# 0.3.10.2 (HOTFIX) — UX/UI: zoom mínimo adaptativo + selección interior + layout persistente

## Resumen
Hotfix de UX/UI con tres objetivos: (1) evitar zoom-out ridículo, (2) facilitar la selección con click dentro de figuras cerradas, y (3) persistir layout de la UI.

## Cambios
- **Zoom mínimo adaptativo**: el mínimo de zoom ahora se calcula según el tamaño de la hoja vs el viewport, con un piso de seguridad.
- **Selección interior**: en figuras cerradas, un click dentro del contorno también selecciona el objeto (hit-test por relleno alfa casi-cero).
- **UI persistente**: se guardan/restauran geometría y estado del `MainWindow` (dock/toolbar).
- **Biblioteca**: panel con splitter (árbol arriba, thumbnails abajo) ajustable y persistente.

## Configuración
- `RCS_CANVAS_HIT_FILL=0/1` (default 1): desactiva/activa la selección interior por relleno.

## Pruebas mínimas
1. Abrir la app y cargar un proyecto.
2. Insertar una figura cerrada y probar selección: click sobre línea y click dentro.
3. Ajustar el zoom (rueda/slider) y verificar que el mínimo se sienta razonable.
4. Ajustar el splitter de Biblioteca, cerrar y reabrir: debe recordar el tamaño.

## Notas
- No cambia el schema `.RCS`.
