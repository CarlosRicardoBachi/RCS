# 0.3.10.2.4 (HOTFIX) — SVG invisible: fallback seguro + selección fácil

## Síntoma
- Doble click en un SVG de Biblioteca inserta el objeto (aparece el **rectángulo de selección**) pero la **figura queda invisible** (pixmap 100% transparente).

## Causa probable (trabajo con hipótesis)
- En algunos entornos (Qt6/PySide6 + driver), el pipeline de preview que usa:
  1) render SVG con `QSvgRenderer` → `QImage`.
  2) extraer el alfa y estilizar (stroke/halo) para contraste.

  puede fallar silenciosamente en la extracción de alfa o en el estilizado y terminar generando una imagen completamente transparente. El objeto existe en escena, pero no hay píxeles para ver.

## Fix
1) **Render robusto**: `QSvgRenderer.render()` se llama con un target rect (`QRectF(0,0,w,h)`) para evitar casos donde el `viewBox` o el defaultSize desplazan el dibujo fuera de la imagen.
2) **Safety net anti-invisible**: si el resultado del estilizado queda sin píxeles visibles (alfa vacío), se hace **fallback automático al render crudo** (la figura se ve aunque no tenga el estilo de halo).
3) **Selección más fácil**: los items raster insertados pasan a `BoundingRectShape` para poder seleccionar/mover clickeando en cualquier parte del objeto (no solo sobre la línea).

## Nota
Este hotfix es deliberadamente conservador: prioriza **visibilidad + UX**. El control fino del estilo (grosor/color/halo en lienzo y miniaturas) se deja para un bloque siguiente una vez estabilizado el render.
