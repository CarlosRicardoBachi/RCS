# 0.3.10.2.72 — HOTFIX GMPR Base SVG Visible

## Problema
El GMPR importaba raster, pero el SVG embebido quedaba invisible (fill negro sobre fondo oscuro, stroke demasiado fino). Sin ese “fondo de referencia” no se puede validar alineación ni ajustar transforms.

## Cambio
En `rcs/ui/canvas_view.py`, el render del SVG base del GMPR ahora:
- Inyecta CSS en memoria para forzar **alto contraste**: `fill:none` + `stroke` claro.
- Deriva `stroke-width` por la escala de rasterización para que sea visible (sin depender del viewBox).
- Ajusta opacidad del fondo por env (`RCS_GMPR_BASE_OPACITY`).

## Config (opcionales)
- `RCS_GMPR_BASE_OPACITY` (0.05..1.0) — default `0.55`.
- `RCS_GMPR_BASE_STROKE_PX` (0.1..10.0) — default `1.25`.
- `RCS_GMPR_BASE_STROKE_USER` — fuerza stroke en unidades del SVG (override).
- `RCS_GMPR_BASE_STROKE_COLOR` — default `#ffffff`.

## Riesgo
Bajo: el CSS solo se aplica al *preview* del fondo GMPR. No altera el SVG en disco ni el modelo del proyecto.

## Test manual
1) Abrir `.GMPR` con SVG embebido.
2) Verificar que el contorno del SVG se ve (alto contraste).
3) Verificar que el raster se puede comparar visualmente contra el contorno.
