# 0.3.10.2.47 — HOTFIX MainWindow Default Canvas Fallback

## Problema
En algunos merges/overrides de patches, `rcs/ui/main_window.py` puede quedar sin el método
`_apply_default_canvas_mm_to_project`, causando crash en el arranque:

`AttributeError: 'MainWindow' object has no attribute '_apply_default_canvas_mm_to_project'`

## Fix
- Se agrega un guard en `__init__`:
  - Si existe el método, se usa.
  - Si no existe, aplica un fallback mínimo leyendo `ui.canvas.default_canvas_mm` desde `rcs_settings.json`.

## Nota
Este hotfix es defensivo: no cambia el flujo normal cuando el método existe; solo evita el crash.
