# 0.3.10.2.13 (HOTFIX) — Canvas: grosor de preview vuelve a responder al menú

## Problema
Con *fit-to-view* (zoom < 1) el canvas estaba aplicando la compensación de grosor como `base / z`.
Eso hace que el grosor crezca al alejarse (zoom-out) y termine saturando en el máximo.
Efecto: líneas demasiado gruesas por defecto y el menú **Ver → Vista previa → Grosor de línea** parecía no cambiar nada.

## Solución
La compensación ahora **solo reduce** el grosor cuando se hace zoom-in:
- si `z > 1`: grosor efectivo ≈ `base / z`
- si `z <= 1`: grosor efectivo = `base`

Así evitamos la saturación en zoom chico y el menú vuelve a tener impacto visible en el lienzo.

## Notas
- No se toca el pipeline de thumbnails (quedan como en 0.3.10.2.12).
- Se ajusta el default de `canvas_stroke_thick` a 2 (AppSettings) para un arranque menos agresivo.
