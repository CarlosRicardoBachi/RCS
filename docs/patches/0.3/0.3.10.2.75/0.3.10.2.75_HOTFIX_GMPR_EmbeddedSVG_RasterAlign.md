# 0.3.10.2.75 — HOTFIX GMPR Embedded SVG + Raster Align

## Síntomas
- Al importar un `.GMPR`, el SVG embebido no se veía (solo aparecía el raster, o se veía “vacío”).
- El raster quedaba fuera de lugar / escala incorrecta, especialmente cuando el SVG tiene `viewBox` con `min-y` negativo (caso típico de exportadores).

## Causa raíz
- El import GMPR solo creaba objetos `raster` y dependía de un preview “base SVG” opcional.
- Los `transform.x/y` de GMPR están en *SVG user-units* (coordenadas del `viewBox`) y requieren restar `min-x/min-y` del `viewBox` antes de convertir a mm.
- Los `transform.sx/sy` de GMPR representan *user-units por pixel* (uu/px), no “factor relativo”, así que la conversión a escala RCS debía usar mm/uu.

## Fix (qué cambia)
- Se materializa el SVG embebido y se inserta como objeto real: `GMPR_BASE_SVG` (`ObjectType.SVG`) en (0,0).
- Se corrige la conversión del raster:
  - `x_mm = (x - vb_min_x) * mm_per_u_x`
  - `y_mm = (y - vb_min_y) * mm_per_u_y`
  - `scale = (sx * mm_per_u) / RCS_MM_PER_PX`
