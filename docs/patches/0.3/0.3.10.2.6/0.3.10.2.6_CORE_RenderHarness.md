# 0.3.10.2.6 (CORE) — Render harness (render_debug) sin UI

## Objetivo
Tener un **harness reproducible** para diagnosticar en segundos los problemas del Bloque R (Render/Geometría), sin abrir la app:
- SVG que queda **invisible** (alpha vacío / stylize rompe),
- **divergencia** entre el estilo del canvas vs thumbnails,
- métricas de render (bbox/alpha) como evidencia para hotfixes.

Esto reduce el “debug por intuición” y da evidencia concreta antes de tocar el pipeline.

## Qué incluye
### Comando
- `python -m rcs.tools.render_debug <svg|carpeta> [--recursive]`

### Outputs (por SVG)
- `*_raw_qtsvg.png` — render crudo QtSvg a cuadrado.
- `*_thumb_style.png` — aplica el stylize de miniaturas (`ThumbCache`).
- `*_canvas_style.png` — aplica el stylize del canvas (incluye fallback anti-transparencia).
- `_summary.json` — por archivo: `alpha_nonzero`, `alpha_bbox`, parámetros (`size`, `scale`, `theme`).

## Cómo usar (rápido)
Ejemplos:
- Un SVG:
  - `python -m rcs.tools.render_debug C:\ruta\a\algo.svg --out C:\tmp\rcs_dbg`
- Una carpeta (recursivo):
  - `python -m rcs.tools.render_debug C:\svgs --recursive`

Vars opcionales:
- `RCS_DBG_SIZE` (default 256)
- `RCS_DBG_RENDER_SCALE` (default 2)
- `RCS_DBG_THEME` (default dark)
- `RCS_DBG_MODES` (default raw,thumb,canvas)
- `RCS_DBG_SLEEP` (default 0)

## Notas técnicas
- El harness **no modifica** el flujo de render normal; es opt-in.
- El modo `canvas` importa `rcs.ui.canvas_view` solo cuando se pide (import perezoso).
