# 0.3.10.2.44 — CORE — NormalizeViewport Runtime Truth

## Objetivo
Que **canvas** y **thumbs** compartan la misma normalización de viewport para reducir diferencias de encuadre/visibilidad.

## Qué se agregó
### Canvas (QtSvg preview)
- `_render_svg_qt()` intenta aplicar `normalize_svg_viewport(svg_path)`.
- Si el contrato devuelve `viewbox=[x,y,w,h]` válido, se aplica con `QSvgRenderer.setViewBox()`.
- El cálculo de tamaño del render prefiere `viewBoxF()` (si está) sobre `defaultSize()`.
- Si se inyecta CSS para stroke (`stroke_px`), se reaplica el viewBox normalizado al renderer "parcheado".

### Thumbs
- Se mantiene el seteo de viewBox por normalización.
- Se agrega log “una vez por archivo” (mismo formato que canvas).

### Telemetría (no intrusiva)
Nuevo helper:
- `rcs/geom/viewport_runtime_log.py` con `log_normalize_once(svg_path, meta, where=...)`.
- Loggea units/ppi/doc/viewbox **una sola vez por SVG** para cazar casos reales sin spamear el log.
