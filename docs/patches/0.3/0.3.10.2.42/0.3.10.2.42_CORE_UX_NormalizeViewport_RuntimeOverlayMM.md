# 0.3.10.2.42 — CORE+UX: NormalizeViewport runtime wiring + overlay mm

## Objetivo
- Reducir divergencias entre **canvas** y **miniaturas (thumbs)** usando el mismo contrato `normalize_svg_viewport()`.
- Mostrar en el lienzo una referencia rápida de dimensiones del canvas en **milímetros**, estable con el zoom.

## Cambios
### 1) Thumbs (QtSvg)
- `rcs/svg/thumbs.py`
  - Intenta aplicar `normalize_svg_viewport(svg_path)` antes de calcular escala.
  - Si hay `viewbox` normalizado válido, se hace `renderer.setViewBox(...)`.

### 2) Canvas previews (QtSvg)
- `rcs/ui/canvas_view.py`
  - `_render_svg_qt()` intenta normalizar y setear viewBox antes de decidir el tamaño base del render.

### 3) Overlay mm en el lienzo
- `rcs/ui/canvas_view.py`
  - En `drawForeground()` se dibuja una etiqueta en esquina superior izquierda:
    - `ANCHO x ALTO mm`
  - Se dibuja en coordenadas de viewport (px) para que no cambie con el zoom.

## Verificación rápida
- Abrir app y verificar:
  - El overlay muestra dimensiones correctas del canvas del proyecto.
  - Thumbs y canvas encuadran de forma consistente en SVGs “problemáticos”.
