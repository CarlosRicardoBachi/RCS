# 0.3.10.2.2 (HOTFIX) — Qt6/PySide6: alfa estable al insertar SVG

## Problema
En algunas instalaciones con **PySide6/Qt6**, `QImage` no expone `alphaChannel()` y/o `setAlphaChannel()`. Esto rompía:
- el **preview** estilizado (stroke/halo) al insertar SVG,
- el **hit-test por relleno alfa** (click dentro del contorno),
mostrando el error: `'PySide6.QtGui.QImage' object has no attribute 'alphaChannel'`.

## Solución
- Se agrega un helper `_extract_alpha8()` que obtiene la máscara alfa como **Alpha8**:
  - Si existe `alphaChannel()` (Qt5) se usa.
  - Si no, se extrae manualmente desde ARGB32 leyendo el byte alfa por pixel (Qt6-safe).
- Se agrega `_tint_from_alpha8()` para crear capas coloreadas a partir de una máscara alfa **sin** usar `setAlphaChannel()`.
- El hit-fill interior ahora respeta `bytesPerLine()` (alineación) tanto en Alpha8 como en ARGB32.
