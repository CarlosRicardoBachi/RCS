# 0.3.10.2.70 — HOTFIX GMPR RasterScale + BaseSVG Visible

## Objetivo
Corregir el P0 de importación GMPR donde el raster embebido aparece **gigante** (por mala interpretación de `sx/sy`) y donde el SVG base embebido parece **inexistente** en canvas oscuro.

## Qué cambia

### 1) GMPR Import — escala de raster
- `sx/sy` se interpretan como **factores relativos** (dimensionless) sobre el base `RCS_MM_PER_PX = 25.4/96`.
- Se agrega una heurística de compatibilidad: si `sx` parece venir como **mm/px absoluto** y eso hace que el raster se vaya a la estratósfera, se convierte a factor relativo.

### 2) GMPR Save — escritura consistente
- Al guardar GMPR, `sx/sy` se escriben como **factores relativos** (y `s` queda como legacy).
- Se preservan `flip_h/flip_v` como flags explícitos.

### 3) Base SVG embebido — preview visible
- Al renderizar el SVG base embebido se inyecta un `<style>` **solo en memoria** para forzar trazos claros (no modifica el SVG persistido).
- La opacidad del base preview se puede ajustar con:
  - `RCS_GMPR_BASE_OPACITY` (default: `0.35`)

## Prueba rápida (manual)
1. Abrir `proyectoabc.GMPR`.
2. Verificar que:
   - El raster embebido queda dentro del canvas (no gigante).
   - El SVG base se ve como preview (aunque sea tenue), alineado con el canvas.
3. Mover un poco el raster y guardar GMPR:
   - Re-importar el GMPR guardado y confirmar que la escala se mantiene.
