# 0.3.10.2.58 — HOTFIX: Preview rerender guard (startup crash)

## Contexto
En `MainWindow._build_ui()` se aplica `CanvasView.set_preview_style(...)` al iniciar. Ese método pide un rerender de previews SVG para que el cambio de estilo (stroke/outline) se vea instantáneo.

En algunos builds/ramas, `CanvasView` no expone `_rerender_svg_previews()`. Resultado: **crash al iniciar** con:
`AttributeError: 'CanvasView' object has no attribute '_rerender_svg_previews'`.

## Cambio
- `CanvasView.__init__()` agrega un **fallback compat**: si `_rerender_svg_previews` no existe, se define como no-op (y se loggea 1 línea).
- `CanvasView.set_preview_style()` ahora **guarda** la llamada: usa `getattr(..., None)` + `callable()`.

## Criterio de aceptación
1) La app inicia sin crash aun si `_rerender_svg_previews` no existe en esa variante de `CanvasView`.
2) Cambiar preview style (stroke/outline) no provoca excepciones.
3) En ramas donde `_rerender_svg_previews` sí existe, el comportamiento visual no cambia.
