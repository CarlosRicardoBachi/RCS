# 0.3.8 — Bloque 3F (Fit to Content)

Fecha: 2026-01-16

## Problema
Algunos SVG tienen **márgenes internos** (espacio en blanco) por el `viewBox`/bounding box original.
Eso rompe el “alineado visual” y el usuario siente que el objeto está “corrido”.

## Solución
Se agregó un botón **Ajustar al objeto** que recorta el render/preview al contenido y persiste un flag (p.ej. `svg_fit_content`) para reabrir igual.

## Pruebas manuales
- Importar SVG con margen inferior.
- Ejecutar Ajustar al objeto.
- Ver que el marco se ajusta a la figura y no al viewBox original.

## Notas
Esto es una solución **de ensamble** (visual/geométrica). No reemplaza “arreglar el SVG” si luego se necesita un path exacto para export.
