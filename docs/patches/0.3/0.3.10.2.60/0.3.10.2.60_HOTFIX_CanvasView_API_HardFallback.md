# 0.3.10.2.60 — HOTFIX CanvasView API HardFallback

## Objetivo
Evitar que el arranque de la app crashee cuando `MainWindow` encuentra un `CanvasView` con API incompleta (parcialmente parcheada), especialmente faltando:
- `CanvasView.set_project`
- `CanvasView._rerender_svg_previews`

En los reportes se vio:
- Warning: preview rerender deshabilitado
- Crash en `main_window.py` por `AttributeError: CanvasView has no attribute set_project`

## Cambios
### 1) `rcs/ui/canvas_view.py`
- Se reemplaza el bloque de “sanity check” por una rutina que **intenta auto-reparar** primero.
- Si faltan métodos críticos, se instalan **fallbacks** minimalistas:
  - `set_project`: guarda `_project`, resetea flags de vista si existen y llama `_rebuild_scene_from_project()` si está disponible.
  - `_rerender_svg_previews`: rerender por objeto si existe `_rerender_svg_preview_for_object`.
- Luego de instalar fallbacks, se re-verifica la API y solo entonces se reporta error.

### 2) `rcs/ui/main_window.py`
- Las llamadas a `self._canvas.set_project(...)` ahora son **safe**:
  - Si existe `set_project`, se usa.
  - Si no existe, se asigna `_project` y se intenta `_rebuild_scene_from_project()`.

### 3) Documentación
- `docs/CHANGELOG.md`: nueva entrada `0.3.10.2.60`.
- `docs/patches/index.md`: se agrega `0.3.10.2.60` (y se referencia `0.3.10.2.59` si ya estaba aplicado).
- `ai/context.json`: versión y “recent_patches” actualizados.

## Criterios de aceptación
1. `python -m rcs.app` inicia sin crash aun si el `CanvasView` viene incompleto.
2. En un proyecto, la UI se carga y el canvas queda vinculado al `Project`.
3. No se rompe la funcionalidad de snap/preview por este hotfix (si hay faltantes, se degrada con fallback, no con crash).
