## Decisiones registradas (para no discutir de nuevo en 2 días)
### Texto editable vs convertido
- Mientras sea **texto editable**, se mantiene como `type="text"` con `text_payload`.
- Se agrega una acción explícita **Convertir a curvas**:
  - genera contornos (paths) como objetos normales
  - conserva metadatos del texto original para poder “reconstruir” (o al menos auditar) qué se generó

### Diacríticos (ñ / tildes / puntos)
Las marcas pequeñas tienden a quedar como islas frágiles. Para el flujo de llaveros:
- modo recomendado: permitir tratarlas como **grabado** (objeto/capa aparte) en vez de corte,
- el usuario decide (por objeto o por conversión).

### Placa/contorno (estilo llavero)
Herramienta prevista: **Texto→curvas→offset exterior** (grosor configurable y uniones redondeadas).
- El agujero/oreja del llavero se modela como objeto aparte (no se infiere automáticamente).
