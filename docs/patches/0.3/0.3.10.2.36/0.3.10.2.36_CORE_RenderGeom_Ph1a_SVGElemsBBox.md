# 0.3.10.2.36 — CORE — Render/Geom Phase 1a (svgelements bbox)

## Objetivo
Arrancar Render/Geom *de forma controlada*: agregar una segunda fuente de verdad geométrica (opcional) para comparar contra QtSvg **sin tocar el runtime**.

Esto permite responder preguntas tipo:
- "¿El viewBox que QtSvg dice coincide con el bbox real del documento?"
- "¿Se me volvió invisible por viewBox raro / unidades / reify?"

## Qué cambia
- Nuevo módulo `rcs/geom/svgelements_bbox.py`:
  - Calcula bbox del documento usando `svgelements` **si está instalado**.
  - Si no está instalado: devuelve `available=false` + error *sin romper nada*.
- Harness `python -m rcs.svg.render_debug`:
  - Añade notas por SVG con:
    - `qtsvg_viewbox` (lo que reporta QtSvg)
    - `svgelements` (bbox y metadatos si está disponible)

## Importante
- **No** se agrega dependencia obligatoria.
- La app no cambia comportamiento.
- Esto es *instrumentación* para avanzar con Skia/svgelements sin navegar a ciegas.

## Archivos
- `rcs/geom/__init__.py`
- `rcs/geom/svgelements_bbox.py`
- `rcs/svg/render_debug.py`

## Prueba rápida (30s)
1) (Opcional) Instalar `svgelements`:
   - `python -m pip install svgelements`

2) Ejecutar harness (modo raw alcanza):
   - `python -m rcs.svg.render_debug componentes/figuras --recursive --modes raw --out _render_dbg --size 256 --scale 2`

3) Abrir un `*_summary.json` en `_render_dbg/...`:
   - Debe aparecer en `results[].notes`:
     - `qtsvg_viewbox: [x,y,w,h]`
     - `svgelements.available: true` y `svgelements.bbox: [x0,y0,x1,y1]`
   - Si no instalaste svgelements: `svgelements.available: false` y `svgelements.error` con el motivo.
