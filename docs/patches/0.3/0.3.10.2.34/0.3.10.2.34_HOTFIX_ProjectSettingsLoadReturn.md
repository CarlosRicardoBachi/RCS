# 0.3.10.2.34 — HOTFIX — ProjectSettings (load_project_settings) + CanvasPrefs ENV sync

## Problema
En 0.3.10.2.33, `MainWindow` asumía que `load_project_settings()` devolvía `(settings, path)`. En realidad, `load_project_settings()` devuelve **solo** el dict de settings. Resultado: crash al iniciar:

- `ValueError: too many values to unpack (expected 2)`

## Fix
- `MainWindow` deja de desempacar el retorno de `load_project_settings()` en todos los puntos.
- Para aplicar el canvas por defecto a proyectos nuevos, se toma primero de ENV (`RCS_CANVAS_DEFAULT_W_MM/H_MM`) y, si no existe, se cae a settings.
- `_update_project_settings(...)` ahora llama `apply_project_settings(...)` luego de guardar para sincronizar ENV en caliente. Esto hace que los cambios de CanvasPrefs afecten a `Archivo → Nuevo` en la misma sesión.
- Feedback: al togglear scroll H/V se actualiza el status inferior.

## Prueba rápida (30s)
1. `python -m rcs.app` inicia sin traceback.
2. F10 → setear tamaño de página (por ejemplo 500x300) → OK.
3. Archivo → Nuevo → el proyecto nuevo arranca con ese canvas (sin reiniciar).
4. Ver → Scrollbars → togglear H y V, mirar el status inferior.
5. Cerrar y reabrir: se mantienen los valores.
