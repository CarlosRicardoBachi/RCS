# 0.3.10.2.35 — `CORE` — CanvasPrefs reset + scroll OFF/AUTO/ON

## Qué resuelve
- Resets de preferencias persistentes sin tocar `rcs_settings.json` a mano.
- Scrollbars con 3 políticas por eje (OFF / AUTO / ON).
- Fix de guardado “en caliente” de settings: se elimina un `TypeError` que podía aparecer al usar acciones que actualizan settings en tiempo real.

## Cambios
### 1) Scroll de lienzo: políticas por eje
En **Ver → Scroll del lienzo** ahora tenés submenús **Horizontal** y **Vertical** con 3 estados:
- **OFF**: `Qt.ScrollBarAlwaysOff`
- **AUTO**: `Qt.ScrollBarAsNeeded`
- **ON**: `Qt.ScrollBarAlwaysOn`

Se persiste en el proyecto en:
- `ui.canvas.scroll_h_policy` / `ui.canvas.scroll_v_policy` con valores: `off | needed | on`.

Compatibilidad:
- Si existen los booleanos viejos `ui.canvas.scroll_h` / `ui.canvas.scroll_v`, se siguen interpretando (True→AUTO, False→OFF), pero si está la policy nueva, gana la policy.

### 2) Resets de CanvasPrefs
Se agregan acciones para volver a defaults:
- **Lienzo → Restaurar página por defecto (app)**
  - Limpia `ui.canvas.default_canvas_mm` y aplica el default de la app.
- **Ver → Restaurar vista de inicio**
  - Limpia `ui.canvas.start_view`.
- **Ver → Restaurar rango de zoom**
  - Vuelve a `ui.canvas.zoom_range = 1.0`.

Todas las acciones muestran feedback en la statusbar.

### 3) Fix de update de settings
Se corrige el llamado interno al aplicar settings luego de guardar:
- Antes: se llamaba `apply_project_settings(settings)` (firma incompatible).
- Ahora: se llama `apply_project_settings(prefer_env=False)`, releyendo de disco y forzando ENV coherente.

## Notas técnicas
- Se agrega `CanvasView.scrollbars_policy()` y `CanvasView.set_scrollbars_policy()`.
- Se agrega `CanvasView.clear_startup_view_state()` para poder “desarmar” una vista guardada en la sesión.
