# 0.3.10.2.3 (HOTFIX) — Biblioteca: inserción visible en Qt6/PySide6

## Síntoma
- Doble click en un SVG de la Biblioteca no inserta nada en el lienzo. El usuario sólo ve el rectángulo de selección (rubber-band) al interactuar con el canvas.

## Causa raíz (confirmada por logs)
- En entornos Qt6/PySide6 faltan APIs históricas de Qt5:
  - `QImage.alphaChannel()` / `QImage.setAlphaChannel()`
  - En algunos builds, el buffer de `QImage.bits()` no expone `setsize()` (o su comportamiento cambia).
- Resultado: al construir máscaras de alfa para el "hit-fill" / estilizado de previews, se levantan excepciones y la inserción se interrumpe antes de crear el item gráfico.

## Cambios
- `rcs/ui/canvas_view.py`
  - Extracción de alfa (`_extract_alpha8`) y tintado (`_tint_from_alpha8`) endurecidos para Qt6: sin depender de `alphaChannel` ni de `setsize()`.
  - Inserción desde Biblioteca: si falla la rasterización, se inserta un placeholder rojo con tooltip de error en lugar de no insertar nada.
  - Inserción más segura: el objeto se agrega al Project sólo después de crear el item gráfico.

- `rcs/svg/thumbs.py`
  - Estilizado de thumbnails Qt6-safe (sin `alphaChannel` / `setAlphaChannel`).
