# 0.3.10.2.3 — HOTFIX: selección fácil + thumbnails Qt6-safe

## Problema
1) **Selección difícil en el lienzo**: objetos insertados como pixmap (preview raster) usan por defecto *hit-test por máscara alfa*, así que si el SVG tiene contornos finos y mucho transparente, cuesta clickear para seleccionar/mover.
2) **Miniaturas oscuras / sin estilizado**: en Qt6/PySide6 no se puede usar `QImage.alphaChannel()` / `setAlphaChannel()`. Eso hacía que el “stylize” (tinta + contorno) falle y termine mostrando la imagen cruda (oscura) en la Biblioteca.
3) **Versión no visible**: el zip previo aplicó la lógica, pero el número de versión quedó sin bump, dificultando confirmar el hotfix.

## Solución
- **Canvas**: el item raster (`QGraphicsPixmapItem`) pasa a ser clickeable por **BoundingRect** (por defecto) para mejorar selección/movimiento.
  - Toggle por env: `RCS_CANVAS_PICK_BOUNDING=0` vuelve al comportamiento por máscara alfa.
- **Thumbs**: extracción/tinte del canal alfa reescritos para Qt6-safe usando `bytesPerLine()` y lectura directa del buffer.
  - Se bump-ea la firma de estilo (`v2`) para invalidar cache y regenerar miniaturas.
- **Versión**: bump a **0.3.10.2.3** (título y `APP_VERSION`).

## Pruebas manuales
- Abrir app y verificar título: `RusticCreadorSvg v0.3.10.2.3`.
- Doble click en `componentes/figuras/*.svg` → aparece y queda visible/seleccionado.
- Selección: click *en cualquier zona del bounding* del objeto → se selecciona fácilmente y se puede arrastrar.
- Biblioteca: las miniaturas se ven con **tinta + contorno** (no “oscuras”).
