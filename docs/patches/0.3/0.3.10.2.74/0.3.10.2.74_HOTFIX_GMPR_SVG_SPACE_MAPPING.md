# 0.3.10.2.74 — HOTFIX GMPR Import: SVG space mapping

## Síntoma
- Importar un `.GMPR` abría el raster pero **sin SVG embebido visible**.
- El raster quedaba con **posición/escala incorrectas**, porque Rustic guarda el transform en **unidades del SVG post-viewBox**.

## Fix
- En `rcs/core/gmpr_io.py`:
  - Al abrir GMPR, se parsea el SVG embebido (width/height + viewBox) para obtener:
    - `canvas_mm` del proyecto (tamaño físico del SVG).
    - factores `mm_per_u_x/y` para convertir user-units→mm.
  - El transform del raster se convierte:
    - `x_mm = x_u * mm_per_u_x`
    - `y_mm = y_u * mm_per_u_y`
    - `scale_x/y` se interpreta como u/px y se convierte a mm/px antes de derivar el factor interno de RCS.
